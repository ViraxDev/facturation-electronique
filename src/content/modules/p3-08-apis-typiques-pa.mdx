# APIs typiques d'une PA : REST, webhooks

<APISequence client:visible />

Les Plateformes Agréées exposent à leurs clients (ERP, portails, SaaS) des **APIs REST** pour le dépôt, la consultation et le suivi des factures. Si chaque PA personnalise son offre, un **tronc commun** se dégage, conforme aux spécifications externes AIFE. Ce module décrit les patterns récurrents, l'authentification, les webhooks et les bonnes pratiques d'intégration.

## 1. Endpoints typiques

Une API PA propose habituellement les opérations suivantes (les chemins varient, l'intention est la même) :

| Verbe | Endpoint | Rôle |
|-------|----------|------|
| POST | `/v1/invoices/send` | Soumettre une facture (UBL, CII ou Factur-X) |
| GET | `/v1/invoices/{id}` | Récupérer une facture déposée |
| GET | `/v1/invoices/{id}/lifecycle` | Lister les statuts d'une facture |
| POST | `/v1/invoices/{id}/status` | Émettre un statut (Refusée, Encaissée…) |
| GET | `/v1/invoices?status=Refusée&since=2026-04-01` | Lister selon filtres |
| POST | `/v1/reporting/transactions` | Soumettre un e-reporting (B2C, transfrontalier) |
| GET | `/v1/directory/{siren}` | Rechercher un destinataire dans l'annuaire PPF |
| POST | `/v1/webhooks` | Enregistrer un endpoint pour les notifications |
| GET | `/v1/health` | Sonde de disponibilité |

Les formats d'échange privilégient **JSON** pour les métadonnées et **XML** (UBL/CII) ou **PDF** (Factur-X) pour le payload de facture, souvent transporté en `multipart/form-data` ou encapsulé en base64.
