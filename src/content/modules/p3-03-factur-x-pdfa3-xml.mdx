# Factur-X : PDF/A-3 + XML intégré

<FacturXExploded client:visible />

Factur-X est le format hybride phare de la facturation électronique française. Comprendre son anatomie est essentiel pour tout développeur ou expert-comptable technique amené à générer, valider ou consommer des factures dans l'écosystème français post-2026.

## 1. Anatomie d'un fichier Factur-X

Un fichier Factur-X est **physiquement un PDF**, mais un PDF très particulier :

- Il respecte la norme **ISO 19005-3:2012 (PDF/A-3)**, garantissant sa conservation long terme.
- Il contient un **stream de fichier embarqué** nommé `factur-x.xml` (ou `zugferd-invoice.xml` pour les anciennes versions ZUGFeRD 1.x).
- Il déclare dans ses **métadonnées XMP** la présence et le profil Factur-X.

L'utilisateur final ouvre un PDF classique, lisible à l'œil. L'ERP ou la PA du destinataire extrait le XML et traite les données structurées. C'est le principe de la **double valence**.

## 2. PDF/A-3 : la norme d'archivage permettant l'embedding

PDF/A est une famille de normes ISO dérivées de PDF, conçues pour l'archivage :

- **PDF/A-1** (ISO 19005-1, 2005) : interdit les fichiers embarqués.
- **PDF/A-2** (ISO 19005-2, 2011) : autorise l'embedding mais uniquement d'autres PDF/A.
- **PDF/A-3** (ISO 19005-3, 2012) : autorise l'embedding de **n'importe quel fichier** (XML, CSV, feuilles de calcul…), avec déclaration obligatoire de la relation.

C'est donc PDF/A-3 qui permet à Factur-X d'exister, en embarquant un XML CII tout en restant un document archivable conforme.

Critères PDF/A-3 typiques :

- Polices toutes **intégrées** (embedded).
- Absence de contenus dynamiques (JavaScript, formulaires actifs non déclarés).
- Espaces colorimétriques explicites (ICC profiles).
- Métadonnées XMP présentes et cohérentes.
